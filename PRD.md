# 콘티노트 (ContiNote) — Product Requirements Document

> **버전**: 1.0 (Stage A~C 완료 기준)
> **최종 업데이트**: 2026-05-15
> **용도**: 디자인·기획·외부 협업자 온보딩

---

## 1. 한 줄 정의

> **"악보 사진/PDF를 올리면, 한국 교회 찬양팀 예배 화면용 PPT가 5분 안에 완성되는 AI 콘티 메이커."**

웹: https://contionote.vercel.app

---

## 2. 비전 & 미션

### 비전
한국 교회 찬양팀 사역자가 **매주 토요일 밤 PPT 만들기에 쓰는 1~2시간을 5분으로** 줄인다.

### 미션
- 악보 OCR → 가사 추출 → 콘티 편집 → PPT 생성을 **한 흐름**으로 묶는다.
- 비전공자 사역자가 **설명서 없이 쓸 수 있게** 만든다.
- 무료(개인) + 유료(교회 단체 라이선스)로 **현장 부담 없이 지속 가능**한 구조를 만든다.

## 3. 타겟 사용자

### 3.1. 핵심 페르소나 — "찬양팀장 김 집사"
| 속성 | 값 |
|---|---|
| 나이 | 30~50대 |
| 직군 | 회사원·자영업·교사·전업주부 (사역은 자원봉사) |
| 직책 | 찬양팀장, 청년부 간사, 부목사 |
| 사용 빈도 | 주 1회 (토요일 밤~주일 새벽) |
| IT 능력 | 카톡·유튜브·한컴 가능. 코딩/디자인 X |
| 디바이스 | 노트북 또는 태블릿. **모바일(휴대폰)에서도 가끔 확인** |
| 결제 | 본인 카드 X. 교회 통장으로만 결제 가능 |
| 핵심 고통 | "한 곡 PPT 만드는데 30분. 일주일 5곡이면 2시간 반" |

### 3.2. 보조 페르소나 — "방송팀 박 형제"
- 20~40대. 주일 송출 담당. PPT를 받아서 ProPresenter나 PowerPoint로 띄움.
- 콘티노트가 만든 **.pptx 파일을 그대로 사용**하는 입장.
- 폰트·색상·줄바꿈이 화면에서 잘 보여야 함이 최대 관심사.

### 3.3. 비타겟
- 대형 교회 전담 미디어팀 (이미 ProPresenter·EasyWorship 등 $400+ 소프트웨어 보유)
- 개인 연주자 (개인용 가사장은 별도 도구)
- 영어권/타국가 교회 (한국어 가사·CCLI 컨텍스트 중심 설계)

## 4. 핵심 사용자 흐름

### 4.1. 골든 패스 (Happy Path)
```
1. contionote.vercel.app 접속
2. 악보 파일 업로드 (드래그앤드롭, 최대 12개)
3. [가사 추출하기] 버튼 클릭
4. 좌측 결과에서 곡 제목·섹션 카드 클릭 → 우측 콘티에 자동 추가
5. (선택) 가사 안에서 Enter 두 번으로 슬라이드 분리
6. PPT 폰트·배경 선택
7. [PPT 다운로드 (.pptx)] 클릭
8. .pptx 파일이 다운로드됨 → PowerPoint/Keynote/구글 슬라이드에서 열기
```

### 4.2. 단축 경로 (이미 추출한 곡 재사용)
```
1. 메뉴 → 곡 라이브러리 (자동 누적된 과거 곡)
2. 검색 → [+ 추가] → 콘티 편집창에 즉시 삽입
```

### 4.3. 클라우드 동기화 흐름
```
1. Google 계정 로그인 (선택, 비로그인도 모든 기능 동작)
2. localStorage에 있던 콘티/곡/템플릿 자동으로 클라우드 업로드 (한 번만)
3. 다른 기기 → 같은 계정 로그인 → 동일한 데이터 그대로 표시
```

## 6. 기능 명세

### 6.1. 가사 추출 (AI OCR)
- **입력**: JPG/PNG/PDF 최대 12개, 총 30MB
- **AI**: Gemini 2.5 Flash (responseMimeType: JSON, temperature 0.2)
- **정확도 토글**: ON 시 PDF scale 2배, temp 0.05, 더 신중한 프롬프트
- **출력**: 곡 단위로 분리된 섹션 배열 (verse/prechorus/chorus/bridge/ending)
- **자동 라벨링**: 1절만 있으면 "Verse", 여러 절이면 "Verse 1/2/3..."
- **OCR 학습**: 사용자가 수정한 가사 패턴을 다음 추출에 힌트로 전달

### 6.2. 콘티 편집기
| 블록 타입 | 화면 표현 | PPT 변환 |
|---|---|---|
| `title` | 큰 헤더 (Serif 24px) | 단독 슬라이드 (제목만) |
| `section` | 칩(라벨) + 편집 가능 가사 본문 | 가사 슬라이드(들) |
| `spacer` | 빈 줄 한 칸 | 무시 |
| `slidebreak` | 점선 구분선 | 슬라이드 경계 |
| `memo` | 파스텔 박스(자유 텍스트) | 자유 텍스트 슬라이드 |

**조작**:
- ↑↓ 버튼으로 순서 이동
- ✎ 버튼으로 가사 직접 편집 (contentEditable)
- ✕ 버튼으로 삭제
- Enter 두 번 = 빈 줄 → 슬라이드 분리
- Enter 한 번 = 줄바꿈 (같은 슬라이드)
- 섹션 추가 시 **자동 2줄 그룹핑** (3줄이면 2+1, 1줄이면 1)

### 6.3. PPT 생성
**테마 (6종)**:
| 키 | 표시명 | 배경 | 글자색 | 용도 |
|---|---|---|---|---|
| `black` | 검정 | #000000 | #FFFFFF | 어두운 예배실 (기본) |
| `white` | 흰색 | #FFFFFF | #1F1B16 | 밝은 예배실 |
| `paper` | 종이 톤 | #FAF5EC | #1F1B16 | 따뜻한 분위기 |
| `meadow` | 초원 | Unsplash 풍경 | #1F1B16 | 봄·야외 예배 |
| `cross` | 십자가 | Unsplash 실사 | #1F1B16 | 절기 강조 |
| `bible` | 성경책 | Unsplash 실사 | #1F1B16 | 말씀 중심 예배 |

이미지 테마는 **흰 반투명 레이어 (transparency 35)** 위에 검정 글자를 올려 가독성 확보.

**폰트 (4종)**:
| 키 | 한글 표시 | 실제 폰트 |
|---|---|---|
| `nanum-myeongjo` | 나눔명조 | Nanum Myeongjo |
| `noto-serif-kr` | 본명조 Pro **(추천)** | Noto Serif KR |
| `nanum-square` | 나눔스퀘어 | NanumSquare |
| `noto-sans-kr` | 본고딕 | Noto Sans KR |

**슬라이드 자동 사이징** (16:9 LAYOUT_WIDE):
| 줄 수 | 폰트 크기 | 한 줄 최대 글자 (한글 기준) |
|---|---|---|
| 1줄 | 64pt | 17자 |
| 2줄 | 54pt | 21자 |
| 3줄 | 44pt | 26자 |
| 4줄 | 36pt | 32자 |
| 5줄+ | ❌ 분리 필요 (빨간 알림) | - |

박스 넘침 시 PowerPoint의 `fit: 'shrink'`로 자동 축소.

**저작권 슬라이드 (선택)**:
- ON 시 PPT 마지막에 곡 제목 + CCLI 번호 + 라이센스 라벨 자동 추가
- 기본값 OFF (한국 교회 50%는 CCLI 미가입)

### 6.4. 다른 형식 내보내기 (메뉴에 숨김, 토글로 노출)
- **TXT** — 콘티 텍스트만 (제목은 ━━━ 강조)
- **DOCX** — Word 문서
- **Plain Slides .txt** — 다른 PPT 도구 호환
- **OpenSong .xml** — OpenSong 교회용 SW 호환
- **클립보드 복사** — 카톡·메모 앱 즉시 붙여넣기
- **공유 링크** — URL hash에 콘티 통째 인코딩, 외부 서버 X

### 6.5. 클라우드 동기화 (Supabase)
**테이블 3종** (모두 RLS, `auth.uid() = user_id`):
- `conti_sets` — 저장된 콘티 (jsonb로 songs+doc 통째 저장)
- `songs` — 추출된 곡 라이브러리 (제목+섹션)
- `templates` — 교회 PPT 기본값

**라우팅**:
- 비로그인 → localStorage
- 로그인 → Supabase (같은 API 통과)
- 첫 로그인 시 자동 마이그레이션 (클라우드가 비어있을 때만)

### 6.6. 추가 기능
- **Undo/Redo** — 콘티/곡 변경 50단계, Cmd/Ctrl+Z, Cmd/Ctrl+Shift+Z
- **단축키** — Cmd/Ctrl+S (콘티 모음), Cmd/Ctrl+Enter (추출)
- **공유 링크** — URL #해시에 base64+encodeURIComponent로 콘티 인코딩

### 7.2. 타이포그래피

- `.dropzone` — 1.5px dashed --rule, hover에 accent 강조
- `.toggle` — 커스텀 스위치 (data-on 속성으로 상태)
- `.label` — 작은 대문자 라벨 (1·2·3·4 영역 머리)
- `.h-display` — 히어로 헤드라인 Serif
- `.h-song` — 곡 제목 Serif

### 7.4. 레이아웃 그리드
- **데스크탑**: max-width 1320px, 좌:우 = 0.85fr : 1.15fr, gap 40px
- **모바일** (≤768px): 1단 적층, grid-template-areas로 1→2→4→3 순서 → 1→2→3→4로 보정됨
- **블록 사이 간격**: --gap 변수 (16px / 20px / 32px)

### 7.5. 마스코트
- 병아리 캐릭터 3종 — `done.png` (히어로) / `reading.png` (분석 중) / `idle.png` (빈 콘티)
- 의도: 비전공자한테 친근함, "AI"가 위협적이지 않게

## 8. 비기능 요구사항

### 8.1. 성능
- 첫 페이지 LCP < 2.5s (Vercel Edge 캐시)
- 가사 추출 API: 정확도 OFF ≤ 15s, ON ≤ 45s (Vercel Hobby maxDuration 120s)
- PPT 다운로드: 5곡 콘티 ≤ 3s (클라이언트 사이드 pptxgenjs)

### 8.2. 접근성
- 모든 버튼에 `aria-label`
- 토글에 `role="switch"` + `aria-checked`
- 포커스 링: 2px solid --accent
- 키보드만으로 전체 흐름 가능 (드래그앤드롭은 ↑↓ 대체 제공)

### 8.3. 반응형
- ≥1024px: 데스크탑 2단
- 768~1024px: 변형 1단 (좁은 데스크탑/태블릿)
- ≤768px: 모바일 1단, 히어로 마스코트 크기 축소

### 8.4. 보안
- Supabase RLS — 본인 데이터만 R/W
- Gemini API 키 서버 사이드만
- 악보 이미지는 추출 시점에만 Gemini로 전송, 우리 서버 저장 X

### 8.5. 브라우저 지원
- Chrome/Safari/Edge/Firefox 최신 2버전
- 모바일 Safari·Chrome
- IE/구버전 Edge 지원 X

## 9. 브랜드 보이스

### 9.1. 톤
- **부드럽고 진정성 있게** — 한국 교회 정서 (자랑 X, 도와드림)
- **간결하게** — 사역자 시간 귀함
- **신뢰감 있게** — 마케팅 과장 단어("최고", "혁신", "AI 기반") 사용 X

### 9.2. 단어 선택
| 권장 | 회피 |
|---|---|
| 콘티, 가사, 슬라이드, 찬양팀 | "워십", "프레젠테이션" (외래어 과다) |
| 만들기, 추가하기 | "생성", "import" |
| 한 번만 더, 다시 시도 | "에러", "실패" |
| 함께 사역하시는 | "고객님" |

### 9.3. 마이크로카피 예시

## 부록 A — 한국 교회 컨텍스트 (외부 협업자용)

**찬양팀 구조** (이해 없으면 디자인 미스 발생):
- 인도자: 곡 흐름 결정, 콘티 짜는 주체
- 보컬·악기: 인도자가 만든 콘티 받아서 연습
- 방송팀: PPT 받아서 주일 송출
- 목회자: 가끔 곡 추가/변경 요청

**"콘티"의 의미**:
- **좁은 의미** = 곡 순서 + 가사 슬라이드 (콘티노트가 만드는 것)
- **넓은 의미** = 예배 흐름 전체 (사도신경 → 찬양 → 말씀 → 봉헌 → 축도)
- 콘티노트는 현재 좁은 의미. 백로그의 "예배 순서 자동 빌더"가 넓은 의미.

**한국 찬양 곡 특성**:
- 1·2절 가사가 같은 음표에 위/아래로 쌓여 있음 (stacked lyrics)
- 후렴이 여러 번 반복
- 영어 후렴이 한국어 절 중간에 섞임
- CCLI 번호 표기는 의무 아님 (50% 정도 사용)
